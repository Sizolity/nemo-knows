# Reviewed Ingest Helper MVP

This document describes the third MVP: reviewing a local ingest draft bundle and
producing an apply plan without modifying the maintained wiki.

## Goal

Bridge the gap between generated drafts and accepted wiki edits:

```text
draft bundle -> deterministic review -> apply-plan.md -> human approval -> wiki edits
```

The helper validates the bundle structure and turns model-generated candidate
pages into a reviewable file-by-file plan.

## Command

```sh
go run ./cmd/nemo \
  -review-bundle drafts/actual-use-llm-wiki \
  -out drafts/actual-use-llm-wiki/apply-plan.md
```

Expected output:

```text
drafts/actual-use-llm-wiki/apply-plan.md
```

## Inputs

The reviewed bundle must contain:

```text
source.md
ingest-plan.md
```

`source.md` is the candidate source summary page.

`ingest-plan.md` is the candidate maintenance plan generated by the local model.

## Deterministic Checks

The helper validates:

- `source.md` has YAML frontmatter.
- `source.md` frontmatter uses `kind: source`.
- `source.md` includes `What It Is`, `Summary`, `Key Claims`, and
  `Suggested Links`.
- `ingest-plan.md` has YAML frontmatter.
- `ingest-plan.md` frontmatter uses `kind: topic`.
- `ingest-plan.md` includes `Source Summary`, `Candidate Wiki Pages`,
  `Suggested Links`, and `Review Checklist`.
- Candidate wiki paths are under `wiki/sources/`, `wiki/concepts/`, or
  `wiki/topics/`.
- Candidate wiki filenames are lowercase and hyphenated.
- Candidate wiki paths are extracted only from the `Candidate Wiki Pages`
  section. Paths mentioned in checklists or manual instructions are not treated
  as candidate content pages.

Invalid candidates such as `wiki/index.md`, `wiki/log.md`, and schema files are
rejected as ordinary knowledge-page targets.

The helper also checks the repository wiki tree and labels candidates as:

- `update existing page` when the exact target already exists.
- `create new page` when no close match is found.
- `create new page; possible duplicate of ...` when a same-category existing
  page has overlapping slug tokens.

## Output

`apply-plan.md` contains:

- the reviewed bundle path,
- validation checks,
- candidate wiki page changes with create/update/possible-duplicate status,
- manual steps for applying approved changes,
- reminders to update `wiki/index.md` and append to `wiki/log.md` only after
  accepted edits.

The apply plan is a review artifact, not a wiki page.

## Review Boundary

The command must not write to `wiki/`.

This MVP intentionally stops before applying changes. It gives the human or
agent a deterministic checklist and proposed file targets, then waits for
explicit approval before any maintained wiki content changes.

## Acceptance Criteria

This MVP is successful when:

- `go test ./...` passes.
- `nemo -review-bundle` writes `apply-plan.md`.
- invalid candidate paths cause a clear error.
- real generated bundles can be reviewed without calling the model.
- `wiki/` remains unchanged by the review command.

## Next Step

After this MVP, a future command can apply a reviewed plan in a controlled way.
That command should still require explicit approval and should update wiki pages,
`wiki/index.md`, and `wiki/log.md` as one auditable operation.
